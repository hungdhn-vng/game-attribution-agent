The decisions that shaped it

Every fork we resolved during brainstorming, in order.

1

Demo data

Public datasets + anonymized real + live crawl. Honors the "no PII / no confidential data" rule while keeping the internal-vs-external story real.

2

Platform

AgentBase — code-based container deploy. Full control over crawler, analysis, and LLM orchestration.

3

Hero use case

Backward diagnosis — "is it us or the market?" The most votable, visceral story; showcases the core differentiator.

4

Architecture

Hybrid orchestrator + Evidence Ledger. Multi-module synthesis where every claim must cite ledger evidence — mechanical anti-hallucination.

5

Output

Interactive HTML report with charts. Internal-vs-market overlay + a likelihood×evidence confidence matrix — far more votable than text.

6

Demo subject

Your own Roblox game (real dashboard data, anonymized) + public Roblox-ecosystem signals. The agent discovers the notable movement from the data.

7

Reach

Self-serve, platform-agnostic onboarding. Any team connects data via chat (CSV + Roblox adapters, LLM-assisted column mapping) — no code.

8

Framework (long-term best)

LangGraph on AgentBase. Graph nodes = our modules, ledger = graph state; native multi-turn memory + conditional drill-down, while staying deterministic (we author the graph).

9

Analytical rigor (research-backed)

Causal, not just descriptive. Market → CausalImpact counterfactual; Segment → Adtributor (ratio-KPI root-cause, citable %); Anomaly → change-point + STL; confidence → self-consistency abstention. Multi-agent debate was researched and rejected. Verified across 25 sources.

10

Accuracy over speed → async pipeline + live crawl + visible reasoning shipped

AgentBase hard-closes any request at ~50s, but we chose to never compromise accuracy. So analyze became an async job advanced across poll requests — each slice stays under 50s, the job resumes, and the model's thinking streams live as a stage-by-stage trace. The benchmark now crawls live external data (real SteamCharts history + cited Perplexity web search), and self-consistency runs as concurrent N-sample sampling at full token budget. This replaced the LangGraph StateGraph for analyze with a custom resumable pipeline (its checkpointer model didn't fit the per-request-budget + cooperative-poll loop). Built via subagent-driven development; backend + React "Attribution Console" frontend in parallel. 181 backend tests pass · verified live (start done in 16s).

11

Conversational layer → OpenClaw chat brain + config-by-chat shipped 12 Jun

Instead of re-implementing chat, we composed the two patterns the platform offers: the GAA stays a custom-runtime analysis engine, and an OpenClaw 1-Click instance ("gaa-chat") becomes the conversational front-end. The React console streams chat through OpenClaw's OpenAI-compatible endpoint (unlocked programmatically over its gateway protocol); OpenClaw taught itself a GAA skill and calls the engine, handing the job back via a [[gaa:job_id]] marker the console polls. The killer feature: the admin reconfigures the system by talking — "switch benchmarks to live crawl", "make reports answer in Vietnamese" — landing in a new key-guarded admin API whose settings (SQLite, store→env→default) the engine re-reads per job. No redeploys, no restarts. 214 backend tests · all five demo flows live-verified end-to-end (reports came back in Vietnamese on command).

How it works

Deterministic where it must be (analysis, evidence, confidence), AI where it adds value (intent, column-mapping, narrative). The LLM never invents findings. analyze runs as an async, resumable pipeline advanced across poll requests (each slice <50s) that streams its thinking trace stage by stage until the report is ready.

The build plan

Five TDD implementation plans, all implemented & deployed. The pure-logic core is framework-agnostic; Plan 0 adds the AgentBase shell; Plan 2A upgrades the analytics; a final async-pipeline build (live crawl + thinking trace) was layered on top via two parallel subagent-driven streams.

Plan 0 — AgentBase + LangGraph integration 9 tasks

MaaS LLM client, LangGraph GraphAgent (route/analyze/onboard nodes over the engine), memory checkpointer, SDK main.py, deploy via the installed skills. Governs the platform shell.

Plan 1 — Foundations & data 10 tasks

Canonical metrics schema, ColumnMapping/GameProfile, CSV + Roblox adapters, sqlite ProfileStore. The platform-agnostic data seam.

Plan 2 — Analysis engine 14 tasks

Hypothesis/ledger schema, rule-based evidence quality, MetricsStore, the 4 modules, source interfaces, synthesizer + citation validator, query planner.

Plan 3 — Onboarding, render & demo 10 tasks

Cached crawler, live benchmark/web-signal sources, LLM column-profiler, 4 Plotly charts, self-contained HTML report, demo packaging.

Plan 2A — Analytics rigor 7 tasks

Replaces the naive modules with research-backed methods: Adtributor root-cause, CausalImpact-style counterfactual, change-point + STL, and the self-consistency abstention gate.

Async pipeline + live crawl + thinking trace shipped

Resumable job advanced across polls (each <50s), concurrent N-sample self-consistency at full token budget, live SteamCharts + cited Perplexity crawl, and a streamed reasoning trace — backend + React console built in parallel.

OpenClaw chat integration 13 tasks · shipped

ConfigStore (store→env→default), key-guarded admin actions, per-job dynamic sources, operator prefs in synthesis, idempotent workspace bootstrap over the gateway protocol, /openclaw SSE proxy + admin-mode chat in the console — spec → plan → subagent-driven build → live E2E, in one day.
